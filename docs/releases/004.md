# 004 发布记录

## 目标

在已验证可用的 Clash Verge 自定义节点基础上，提供可直接导入的订阅 URL，减少手动配置节点时复制密码、端口、加密方式出错的概率。

## 变更

- Caddyfile 生成逻辑支持 `CLASH_SUBSCRIPTION_TOKEN`。
- 当订阅 token 已配置时，部署脚本生成私密 Clash YAML 文件，`https://api.zero007.chat/clash/<token>.yaml` 由 Caddy 静态托管返回。
- 订阅内容使用服务器 `.env` 中的 `CLASH_NODE_SERVER`、`CLASH_NODE_PORT`、`CLASH_NODE_METHOD` 和 `CLASH_NODE_PASSWORD` 动态生成。
- 订阅路径使用至少 16 位的私密 token，且只允许字母、数字、`_`、`-`，避免路径注入。
- `clash-node` / `sing-box` 日志校验不再把公网客户端连接失败噪声视为部署 fatal。
- 更新 `docs/clash-node.md`，补充订阅 URL、查看命令和保密边界。

## 验收

- `validate-candidate` 通过。
- 蓝绿部署成功。
- Caddy 订阅路径返回 `200`，内容包含 `proxies`、`proxy-groups` 和 `rules`。
- 原有 `https://api.zero007.chat/health` 继续返回 `200`。
- `sub2api-clash-node` 容器保持 `healthy`。

## 回滚

如需禁用订阅 URL，可从服务器 `.env` 删除 `CLASH_SUBSCRIPTION_TOKEN` 后重新执行部署或切换槽位，使 Caddyfile 回到仅反代应用的配置。
