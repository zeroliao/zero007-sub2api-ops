## 变更内容

- IP 管理中允许删除已绑定账号的代理 IP。
- 删除已绑定代理时，前端二次确认会明确显示绑定账号数，并提示相关账号会取消代理关联。
- 后端删除代理时先清空 `accounts.proxy_id`，再将 active 的 `account_proxy_bindings` 标记为 `proxy_unavailable`，最后删除代理记录。
- 批量删除代理不再跳过已绑定账号的代理，删除后同样由后端统一解绑。
- 被取消关联的账号后续请求时会变成无手动代理状态，由现有代理分发逻辑重新分配可用代理。

## 变更原因

用户要求 IP 管理中可以删除绑定了账号的 IP；删除时需要二次确认，成功后账号和 IP 的关联关系必须被去掉，并让这些账号在下次请求时重新分配 IP。旧逻辑会直接阻止删除已绑定代理，无法满足代理池维护和失效 IP 清理场景。
