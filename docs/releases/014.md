## 变更

- 代理订阅节点页增加独立路由，支持从节点详情返回订阅源列表。
- 节点表展示订阅源解析出的全部节点，以及延迟、纯净度、分数、失败/超时、休眠和最近扫描信息。
- 节点策略支持实时预览筛选结果，不满足条件的节点不可选并展示原因。
- 节点策略保存只提交可编辑字段，降低后端更新接口的兼容风险。
- 本地初始化示例账号 `admin@sub2api.local` 的示例密码设置为 `admin123456`。
- 运维文档补充 Windows PowerShell 中文/UTF-8 输出乱码处理经验。
- 将 compose 已引用的 `deploy/sing-box/config.json` 纳入 ops 基线，避免 Git 驱动部署时缺少挂载文件。
- 追加独立 `vpn-ws-node` sidecar，保留原 Shadowsocks 节点，同时通过 Cloudflare WebSocket TLS 暴露 VLESS/Trojan 节点。
- Clash 订阅增加 `proxy-server-nameserver` 和节点域名 `nameserver-policy`，避免 Clash/Mihomo fake-ip 解析导致代理服务器域名自绕超时。
- VLESS/Trojan 节点补充 `client-fingerprint: chrome` 和显式证书校验配置，提升 Clash Verge/Mihomo 兼容性。
- `vpn.zero007.chat` 仅保留 `/vless` 和 `/trojan` WebSocket 入口，普通页面访问返回 `404`，避免误进入 Sub2API 页面。
- 部署脚本支持自动初始化 `VPN_WS_*` 环境变量，并把 `vpn-ws-node` 纳入镜像 pin 校验、sidecar 健康检查和日志检查。
- 运维脚本增加 `sync-sidecar-proxies` 动作，用于从已选择订阅节点生成 sing-box sidecar 出口并同步 endpoint 状态。
