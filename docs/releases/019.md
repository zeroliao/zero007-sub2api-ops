## 变更内容

- 固化 `dev/<version>`、`release/<version>`、`main` 之间同一批逻辑改动只能沿一条提交链流动的规则。
- 明确同步分支时优先使用 fast-forward。
- 明确 fast-forward 失败时必须先检查分叉原因和重复补丁，再决定合并、删除或归档分支。
- 更新源码发布快捷流程和两个仓库的 `AGENTS.md`，让后续 Codex 执行版本流程时遵循同一规则。

## 变更原因

v018 发布收尾时发现 `sub2api/dev/018` 与 `main` 存在内容等价但 commit hash 不同的分叉。该问题通常由同一批补丁在不同分支重复提交、rebase 或 cherry-pick 造成，会导致 `--ff-only` 同步失败并增加发版收尾复杂度。

本版本将复盘结论写入流程，避免后续版本重复出现同类分叉。
