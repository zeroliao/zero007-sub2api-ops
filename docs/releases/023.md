## 变更内容

- 后端将历史默认 `keymd:AbuseIPDB API Key` 视为未设置自定义引用，扫描节点时会优先使用页面保存的全局 AbuseIPDB API Key。
- 保留真正自定义引用的原有行为，例如 `env:`、`literal:` 和非默认 `keymd:`。
- 前端订阅源列表和节点弹窗状态显示同步兼容旧默认引用，避免把默认引用误显示为“自定义 Key 引用”。
- 增加单元测试覆盖旧默认 `keymd:AbuseIPDB API Key` 使用全局 Key 的行为。

## 变更原因

用户已经在页面配置了全局 AbuseIPDB API Key，但部分订阅源历史上保存了旧默认值 `keymd:AbuseIPDB API Key`。后端此前按“订阅源自定义引用优先”处理，导致扫描时跳过页面全局 Key，继续读取服务器 `key.md`，最终报 `unable to resolve API key label "AbuseIPDB API Key" from key.md`。
