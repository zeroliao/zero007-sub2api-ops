追加修复 2（OpenAI 账号测试代理自愈）：
- source commit: `35f49b2b`
- image tag: `ghcr.io/zeroliao/sub2api:release-002-35f49b2`
- image digest: `ghcr.io/zeroliao/sub2api@sha256:fdd8409826585be8db6dfebec0a6007f4287893f5e2f415b6344370c00d44d0c`
- 背景：新账号被自动分配到失效代理时，OpenAI OAuth 账号测试会在 TLS 握手阶段失败；已连接过且未绑定坏代理的账号仍可能直连成功。
- 行为：OpenAI 账号测试遇到代理/TLS 连接类错误时记录账号-代理绑定失败；连续失败达到阈值后将该绑定标记为 `proxy_unavailable`，优先换用其它 active 代理；无可用代理且 `direct_fallback_mode=global` 时直连兜底。
- 覆盖：OpenAI text / compact / image OAuth 测试路径；非代理业务错误不触发自愈；请求体不可重放时不盲目重试。

追加修复 3（代理自愈日志降噪与极端路径复核）：
- source commit: `6ff5c9b1`
- image tag: `ghcr.io/zeroliao/sub2api:release-002-6ff5c9b`
- image digest: `ghcr.io/zeroliao/sub2api@sha256:4bae24c4c30bd30faa37af56f13ef2a39531c2b29e3c10a6efa26b7dc1eb745d`
- 背景：代理自愈已生效，但标准库 `log.Printf` 被统一接入为 `stdlog` 后，正常的自愈重试提示会显示为 `ERROR stdlog`，容易干扰部署后日志判断。
- 行为：OpenAI 账号测试代理自愈重试日志改为结构化 `slog.Warn`，保留 account_id、from_proxy、to_proxy / direct_fallback、cause 等字段，避免把可恢复路径误判为硬错误。
- 极端路径覆盖：
  - 账号已有失效代理绑定：连续代理/TLS 失败后标记当前绑定不可用，并尝试分配其它 active 代理。
  - 新账号首次被分配到失效代理：同样累计失败并进入换代理或直连兜底。
  - 代理池全部不可用：当 `direct_fallback_mode=global` 时持久切到直连；关闭直连兜底时返回原始连接错误。
  - 非代理错误：上游 401、业务响应异常、普通网络错误不触发代理自愈，避免误伤账号状态。
  - 请求体不可重放：不进行盲目重试，避免重复消费不可复用 body。
  - 成功恢复：代理成功后清零账号-代理绑定失败计数，并将 cooling 代理恢复为 healthy。
